# Philosophy

Git Vibe is for solo developers and teams that want the speed of AI without the sludge of branch chaos.

It is not trying to make Git fancier.

It is trying to make parallel work feel obvious.

## `main` should stay shippable

`main` is not where half-finished experiments go to hide.

It should stay clean enough to release from, review from, and trust.

That only works when unfinished work gets pushed out into its own lanes.

## One task deserves one lane

A worktree is cheap. Confusion is expensive.

When one checkout holds three ideas, two experiments, and an AI-generated diff, nobody feels fast. They feel nervous.

Git Vibe fixes that by making one task equal one worktree.

## AI agents need boundaries

AI is a force multiplier. It is also a mess multiplier when everything happens in the same lane.

Git Vibe treats agents like real collaborators. Give them context. Give them a branch. Give them a safe place to work.

That is what a vibe is.

## Less ceremony, more flow

Most Git workflows add process when the real problem is ambiguity.

Git Vibe goes the other way:

- no `develop`
- no stash juggling
- no mystery branches
- no giant checkout you promise to clean up later

Open a vibe. Do the work. Open the PR. Finish cleanly.

## Cleanup is part of the workflow

A good workflow does not just help you start.

It helps you leave the place clean.

That is why Git Vibe cares about finishing, checking, reopening, diffing, and deleting stale lanes. The point is not just speed. The point is confidence.
